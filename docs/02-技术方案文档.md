### 2.2 蛋白质结构预测模块

#### 技术方案

**方案 A: 使用 AlphaFold2/OpenFold**
- 优点：准确率最高
- 缺点：计算资源需求大 (GPU 16GB+)
- 适用：高精度需求场景

**方案 B: 使用 ESMFold**
- 优点：速度快 (比 AF2 快 10 倍+)
- 缺点：准确率略低
- 适用：快速预览/批量处理

**方案 C: 混合策略**
- 先用 ESMFold 快速筛选
- 对候选结构用 AlphaFold2 精修
- 平衡速度和精度

│  │  任务队列   │  │  GPU Worker │  │  结果缓存   │ │

## 3. 数据库设计

### 3.1 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) DEFAULT 'user', -- user, admin, enterprise
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 项目表
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    organism VARCHAR(100), -- 宿主生物
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- DNA 序列表
CREATE TABLE dna_sequences (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID REFERENCES projects(id),
    name VARCHAR(255) NOT NULL,
    sequence TEXT NOT NULL, -- DNA 序列
    length INTEGER GENERATED ALWAYS AS (LENGTH(sequence)) STORED,
    gc_content DECIMAL(5,2) GENERATED ALWAYS AS (
        (LENGTH(REGEXP_REPLACE(sequence, '[GC]', '', 'g'))::DECIMAL / LENGTH(sequence)) * 100
    ) STORED,
    metadata JSONB, -- 灵活存储额外信息
    created_at TIMESTAMP DEFAULT NOW()
);

-- 蛋白质结构预测任务
CREATE TABLE protein_predictions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    sequence TEXT NOT NULL,
    model_used VARCHAR(50), -- alphafold2, esmfold
    status VARCHAR(50) DEFAULT 'pending',
    result_path VARCHAR(500), -- S3 路径
    confidence_score DECIMAL(5,2),
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);

-- 订阅/计费表
CREATE TABLE subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    plan_type VARCHAR(50) NOT NULL, -- free, pro, enterprise
    status VARCHAR(50) DEFAULT 'active',
    start_date DATE NOT NULL,
    end_date DATE,
    amount DECIMAL(10,2),
    currency VARCHAR(10) DEFAULT 'CNY'
);

-- 索引优化
CREATE INDEX idx_projects_user_id ON projects(user_id);
CREATE INDEX idx_sequences_project_id ON dna_sequences(project_id);
CREATE INDEX idx_predictions_user_id ON protein_predictions(user_id);
CREATE INDEX idx_predictions_status ON protein_predictions(status);
```

### 3.2 数据模型关系

```
users (1) ──────< (N) projects
projects (1) ───< (N) dna_sequences
users (1) ──────< (N) protein_predictions
users (1) ──────< (1) subscriptions
```

## 5. 安全设计

### 5.1 认证与授权

- **JWT Token**: 访问令牌 (15 分钟) + 刷新令牌 (7 天)
- **RBAC**: 基于角色的访问控制
- **API Key**: 用于程序化访问

### 5.2 数据安全

- **传输加密**: HTTPS (TLS 1.3)
- **存储加密**: 敏感字段 AES-256 加密
- **数据隔离**: 多租户数据逻辑隔离
- **备份策略**: 每日增量 + 每周全量

### 5.3 生物安全

- **序列筛查**: 自动检测危险序列 (毒素/病原体)
- **使用审计**: 记录所有设计操作
- **合规检查**: 遵循生物安全法规
